Evidence from `v80_fluence_0p5_100ps_free_surface_ablation_quickcheck.in`:

- With `alat = 5.431 A`, the lateral size is about `19 * 5.431 A = 103.189 A = 10.319 nm`.
- The free-surface heat source is implemented as z slabs:
  - `region heat_01 block INF INF INF INF 288.705000 298.705000 units box`
  - `group heat_01 region heat_01`
  - `variable qheat_01 equal 48775.0064256*v_pulse`
  - `fix heatfix_01 heat_01 heat 1 v_qheat_01`
- The same pattern repeats for z layers 1-10.

This means:

- There is no subtraction of the box center `xc`.
- There is no lateral `dx = x - xc`.
- There is no minimum-image periodic correction in x.
- There is no `w0` parameter in the free-surface heat deposition.
- There is no `gaussian_x` multiplier in the final deposited heat.
- Heat is deposited using current membership in z-layer regions, not an initial-coordinate `x0, z0` absorption map.

The config also does not define a nanometer-scale `w0` for this v80 free-surface case. The heat source is Beer-Lambert in z and uniform over x/y.

## Answers to the requested questions

### 1. Are ejecta atoms from the laser center?

For the requested 0.6 case: not answerable without the atom dump.

For the visible v80 free-surface model structure: the heat source is laterally uniform, so there is no modeled laser center in x/y. If the 0.6 case was generated by the same mechanism, a full-width x0 distribution would indicate a heat-source/model-definition issue, not merely a plotting artifact from using `x_current`.

### 2. Heat-source problem or box too small?

The available v80 free-surface model has a heat-source problem first: no lateral Gaussian is applied.

The lateral box size is also small, about 10.3 nm. If a future Gaussian spot uses a radius of several nm, then the ratio must be checked:

- If `Lx < 4*w0`, the surface will be close to uniformly heated and a clear center crater is unlikely.
- Recommended minimum: `Lx >= 4*w0`.
- Better: `Lx >= 6*w0`.

If the intended box width is about 10 nm, use a validation case with `w0 = 1.5-2.0 nm` and `Lx = 12-16 nm`. For a more morphology-oriented case, use `Lx = 20-30 nm` with `w0 = 3-5 nm`.

### 3. Has a stable crater formed by 100 ps?

For 0.6 J/cm2: not answerable from atom-level evidence because the dump is absent.

For interpretation, keep the distinction strict:

- `ejecta` means high-speed upward atoms.
- `crater` means stable center material removal relative to the surrounding surface.

Do not use a single atom `zmax` as crater depth or plume body height. A paper-level crater figure should show robust center surface depression, not just high-z outliers. With only 100 ps, this is likely still an early ejecta/motion stage; if the heat source is corrected, continuing to 150-200 ps is reasonable only after verifying the 100 ps source localization and surface profiles.

## Minimum modification plan if this model generated the 0.6 case

Priority 1: add a true lateral Gaussian source with consistent metal-unit lengths:

```text
xc = 0.5 * (xlo + xhi)
dx = x - xc
dx = dx - Lx * round(dx / Lx)
gaussian_x = exp(-2 * dx^2 / w0^2)
```

LAMMPS metal length is Angstrom. If `w0` is specified in nm, convert it with `1 nm = 10 A`.

Priority 2: base absorption on initial coordinates:

Use `x0, z0` for the spatial absorption mask so the heating pattern does not drift with moving atoms.

Priority 3: adjust spot/box:

- Case A: keep a roughly 10 nm box, use `w0 = 1.5-2.0 nm`, `fluence = 0.6 J/cm2`, `time = 100 ps`.
- Case B: expand to `Lx = 20-30 nm`, use `w0 = 3-5 nm`, `fluence = 0.6 J/cm2`, `time = 100 ps`.
