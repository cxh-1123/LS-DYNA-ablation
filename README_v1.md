# Version 1 — 硅片激光烧蚀「最小纯热模型」

> 目标：在 LS-DYNA 里验证 **高斯激光热源 + 硅热物性 + 温度场** 是否合理，
> 不做任何复杂物理（无相变、无侵蚀、无 SPH、无 plume、无结构光后处理）。
> 跑出来温度看着合理，再进下一阶段。

## 2. 单位制（重要）

整个 V1 用 LS-DYNA 的 **mm – ms – kg – kN – GPa – K** 一致单位制
（与项目已有的 `test.k` 保持一致，方便后续模块接续）。

| 物理量 | 单位 | 备注 |
| --- | --- | --- |
| 长度 | mm | 500 µm 写作 `0.5` |
| 时间 | ms | 100 ns 写作 `1.0e-4` |
| 质量 | kg | |
| 力 | kN | F = m·a → 1 kg · 1 mm/ms² = 1 kN ✓ |
| 应力 / 弹模 | GPa | kN / mm² = 1 GPa ✓ |
| 温度 | K | 绝对温度 |
| **能量** | **J** | 1 kN · 1 mm = 10³ N · 10⁻³ m = 1 J ✓ |
| **功率** | **kW** | 1 J / 1 ms = 1 kW ✓ |
| **热流密度** | **kW/mm²** | 1 kW / 1 mm² = 10⁹ W/m² ✓ |
| 密度 ρ | kg/mm³ | 硅 2330 kg/m³ = **2.33 × 10⁻⁶** kg/mm³ |
| 比热 cp | J/(kg·K) | 硅 ≈ **700** |
| 热导率 k | kW/(mm·K) | 硅 150 W/(m·K) = **1.5 × 10⁻⁴** kW/(mm·K) |

> **若想换成 mm-ms-tonne-N-MPa**：把 ρ 改成 `2.33e-9 tonne/mm³`，
> cp 改成 `7.0e8 mm²/(ms²·K)`，k 改成 `150 tonne·mm/(ms³·K)`，
> 热流写作 `1.0e7 N/(ms·mm)`。本项目暂不切换，省麻烦。

## 3. 激光能量 → 表面热流的换算（最容易出错的一步）

**几何**：高斯光斑半径 w₀ = 20 µm；脉冲宽度 τ = 100 ns（矩形）。
**目标**：选一个表面峰值热流 I₀，使得纯热模型不会数值爆掉。

### 3.1 关键公式

- 表面热流（W/m²）的空间分布（最常用约定，w₀ 为 1/e² 半径）：

  ```
  q(r, t) = I₀ · exp(-2 r² / w₀²) · f(t)
  ```

- 中心点（r = 0）半无限体解析峰值温升（矩形脉冲，常物性）：

  ```
  ΔT_max ≈ 2 · I₀ · √(α · τ / π) / k
  ```

  其中 α = k / (ρ · cp) 是热扩散率。

- 高斯光斑内总吸收能量：

  ```
  E_abs = I₀ · τ · π · w₀² / 2
  ```

### 3.2 V1 取的具体数

| 量 | 值 |
| --- | --- |
| α | 9.20 × 10⁻⁵ m²/s |
| L_th = √(α·τ) | ≈ 3.0 µm （脉冲期间扩散深度） |
| E_abs | ≈ 0.63 µJ （非常小 — 这是有意保守） |
| ΔT_max | ≈ **228 K** |
| 预期峰值温度 | ≈ **528 K** （仍远低于熔点 1687 K，安全） |

> **注**：解析估计假设半无限体 + 表面均匀常值热流。真实 2D 高斯 + 有限厚度
> 算出来的峰值温度 **会比这个公式低一点**（约 85–95%），因为径向扩散也带走热量。
> LS-DYNA 实测（SMP R16 + 4 CPU）：**中心节点 8081 在 t = 100 ns 时达到 501.5 K
> （ΔT ≈ 201 K）**，与解析估计 528 K 相差 5%。
> 500 ns 时已冷却到 331 K（仍比初温高约 31 K，符合扩散预期）。

### 3.3 单位换算的算式（自己跑一遍 sanity check）

`I₀ = 10 kW/mm²`
等价于
`= 10 × 10³ W / (10⁻³ m)² = 10⁴ W / 10⁻⁶ m² = 10¹⁰ W/m² = 1 GW/cm²`。

`I₀ · τ · π · w₀² / 2`
`= 10¹⁰ × 10⁻⁷ × π × (2 × 10⁻⁵)² / 2`
`= 10¹⁰ × 10⁻⁷ × π × 4 × 10⁻¹⁰ / 2`
`≈ 6.3 × 10⁻⁷ J = 0.63 µJ`。

## 4. .k 文件里每个关键参数的物理意义

`models/v1_thermal_silicon.k` 是由 `scripts/_build_v1_mesh.py` 一次性生成的
（生成后可以独立运行，不需要再调用 Python）。

| 关键字 | 字段 | 物理意义 |
| --- | --- | --- |
| `*CONTROL_SOLUTION` | `SOLN = 1` | **只解热**，不解结构。 |
| `*CONTROL_TERMINATION` | `ENDTIM = 5.0e-4` | 计算总时长 = 500 ns。 |
| `*CONTROL_THERMAL_SOLVER` | `ATYPE=1, PTYPE=0` | 瞬态线性热传导。 |
| | `SOLVER=11` | PCG 迭代求解。 |
| `*CONTROL_THERMAL_TIMESTEP` | `TS=1` | 变步长。 |
| | `TIP=1e-8` | 初始步 10 ns。 |
| | `TMIN/TMAX` | 步长上下界 10⁻²–10² ns。 |
| | `DTEMP=50` | 单步最大温升 50 K（步长自适应判据）。 |
| `*MAT_THERMAL_ISOTROPIC` | `TRO`, `HC`, `TC` | 密度、比热、导热系数。 |
| `*SECTION_SHELL` | `ELFORM=15` | **2D 轴对称**单元（体积加权），Y 为旋转轴。 |
| `*PART` | `MID=0, TMID=1` | 不用结构材料，只用热材料。 |
| `*NODE` | x = r ≥ 0, y = z, z = 0 | 全部节点在 X-Y 平面，旋转轴是 Y。 |
| `*INITIAL_TEMPERATURE_SET` | `NSID=0, TEMP=300` | NSID=0 表示对所有节点设初温。 |
| `*DEFINE_CURVE 1` | (t, I₀(t)) | 矩形脉冲：[0, τ] 为 I₀，之后为 0。 |
| `*BOUNDARY_FLUX_SEGMENT` | `mlc_i = -e^(-2 r²/w₀²)` | **负号 = 热流入 = 加热**，与 LCID 相乘得到 q(r,t)。 |

**网格规模**：(NR+1) × (NZ+1) = 101 × 81 = **8181 节点**；NR × NZ = **8000 单元**。

## 6. 如何判断结果是否合理

跑完之后，按这个顺序检查（**任何一步失败就停下来**，不要直接进 V2）：

### 6.1 自动检查

```powershell
.\.venv\Scripts\python.exe scripts\check_v1_outputs.py
```

脚本会自动：

- 确认 `d3plot / d3hsp / messag` 这些文件存在；
- 在 `messag` 末尾找 `Normal termination`；
- 数 `ERROR / FATAL / WARNING` 个数；
- 在 `d3hsp` 里找几个关键参数的回显；
- 打印一遍 **解析估计的预期峰值温度**（≈ 528 K）。

### 6.2 用 LS-PrePost 看 d3plot

打开 `results\v1\d3plot`：

1. **形状**：温度高点应在**顶面中心 (X = 0, Y = 0.2 mm)**，呈现径向高斯衰减 + 沿厚度方向指数衰减。
2. **峰值**：
   - 脉冲结束瞬间 (t ≈ 100 ns) 中心节点温度 **≈ 501 K**（实测）；
   - t = 500 ns 时（冷却阶段）回落到 **≈ 331 K**。
3. **径向曲线**：在 Y = 0.2 mm 的顶面沿 X 取温度曲线，应该是高斯形（半高宽 ≈ w₀ × √(2 ln2) ≈ 24 µm 量级）。
4. **轴向曲线**：在 X = 0 沿 Y 取温度曲线，从 200 µm（最高）单调降到 0 µm。
5. **底面**：底面 (Y = 0) 中心温度上升应 **远小于 1 K**（脉冲短、热扩散长度只有 ~3 µm，热扩散不到 200 µm 厚度的另一端）。

参考实测时程（项目内已跑通一次的结果）：

| t (ns) | T_max (K) | ΔT (K) | 阶段 |
| --- | --- | --- | --- |
| 0     | 300.0 | 0     | 初始 |
| 20    | 372.0 | 72    | 加热中 |
| 60    | 452.3 | 152   | 加热中 |
| **100** | **501.5** | **201** | **脉冲结束 ← 峰值** |
| 200   | 379.2 | 79    | 冷却中 |
| 300   | 352.6 | 53    | 冷却中 |
| 500   | 331.2 | 31    | 冷却尾段 |

### 6.3 红线（出现下面任何一条都是错）

- `messag` 没有 `Normal termination` → 模型崩了；
- 中心峰值温度 > 1500 K → I₀ 偏大或符号错；
- 中心峰值温度 < 350 K → 多半符号反了（加热变冷却）或单位错；
- 温度高点不在 (0, 0.2) 而在底面 / 边缘 → 边界条件加错了节点。

## 7. 调参速查表（**温度不对**怎么改）

> 顺序很重要：先确认**符号**和**单位**，再调强度。

### 7.1 跑出来温度几乎不变 (< 320 K)

可能问题 | 在哪里改
--- | ---
`mlc` 符号反了，变成冷却 | `*BOUNDARY_FLUX_SEGMENT`，V1 用的是 **负值**。改成正值看是不是反向了。
单位错误（差几个量级） | 重点检查 `cp = 700`、`k = 1.5e-4`、`ρ = 2.33e-6`、`I0 = 10`。
LCID 没生效 | 看 d3hsp 里有没有打印 LCID = 1 的折点。
初温没设上 | d3hsp 应能找到 `INITIAL TEMPERATURE = 300`。

### 7.2 温度爆炸 (>> 1500 K)

调小 | 期望效果
--- | ---
`I0_PEAK = 10 → 5 kW/mm²` | ΔT 直接减半（≈ 110 K）
`TAU_PULSE = 1e-4 → 5e-5 ms` (50 ns) | ΔT ∝ √τ，约 0.71×
`W0 = 0.020 → 0.040 mm` | 同样 I₀ 下，扩散更快、ΔT 略降

### 7.3 想让温度更明显（不建议在 V1 里这么干）

升 | 后果
--- | ---
`I0_PEAK = 10 → 30 kW/mm²` | ΔT ≈ 680 K，峰值温度逼近熔点，纯热模型仍稳定但接近 V1 的可信上界。
`I0_PEAK = 10 → 50 kW/mm²` | 解析峰值 ≈ 1440 K（接近熔点 1687 K），V1 仍跑得动但物理上已不准（没相变）。

### 7.4 网格相关

参数 | 含义 | 默认 | 改的时机
--- | --- | --- | ---
`NR` | 径向单元数 | 100 | 想要更平滑的高斯轮廓 → 增大到 200
`NZ` | 厚度方向单元数 | 80 | 想要解析更短脉冲 / 更陡梯度 → 增大到 160
`DT_PLOT` | d3plot 输出间隔 | 2e-5 ms | 想要更精细动画 → 改到 5e-6 ms

> 改这些参数后，必须重新运行 `scripts\_build_v1_mesh.py` 重生成 .k。

## 8. 已踩过的坑（验证 V1 期间真实遇到，已修好）

> 留作 V2 起步前的备忘。这些都是真坑，每个都让我吃了一次错。

1. **LS-DYNA 不吃非 ASCII 字符** —— 即便是 `$` 注释里的中文也会让 R16 报
   `*** Error 20001` 并退化成 structural-only 分析。`.k` 文件**必须**纯 ASCII。
   构建脚本 `_build_v1_mesh.py` 已强制 `ascii` 编码 + 失败即抛。
2. **PowerShell 同理** —— Windows PowerShell 默认按 ANSI 读 `.ps1`，
   中文会被解码成乱码并破坏语法。`run_v1.ps1` 全 ASCII。
3. **`*CONTROL_THERMAL_TIMESTEP` 第 2、3 字段不是「初始步长，something」**
   而是 `(TIP, ITS)`：
   - `TIP` = **时间积分参数 θ**（0.5 = Crank-Nicolson），不是步长。
   - `ITS` = **初始热步长**（必须 > 0）。
   把这两个搞反，LS-DYNA 会报 `*** Error 11365 No initial time step size`。
4. **`*SECTION_SHELL` 即使是 ELFORM=15（轴对称体），Card 2（厚度）也必须存在**，
   否则 LS-DYNA 会把下一行（例如 *PART 的 title）当成 Card 2 吃掉，
   然后产生几十条 `*** Error 10246 reading *SECTION card` 的连锁错误。
   `T1..T4` 给 `1.0` 占位即可（被忽略）。
5. **`*BOUNDARY_FLUX` 符号约定**：负值代表「热流入物体」，
   所以我们用 `mlc_i = -exp(-2 r²/w₀²)` + 正的 LCID 实现加热。
   配错符号 → 模型变成「冷却」，T_max 只会下降。
6. **`*PART` 的 title 行不能以 `$` 开头**（那样被当成注释跳过，
   下一行的数据卡反而被当成 title）。

## 9. V1 故意不做的事

> 出现在 V2+，不要现在做。

- ❌ `*MAT_ADD_EROSION` —— 不删单元
- ❌ 汽化阈值删除 / 隐式相变 —— 不做
- ❌ SPH —— 不做
- ❌ plume / 气化产物 —— 不做
- ❌ 结构光后处理 —— 不做
- ❌ 温度依赖的 ρ(T)、cp(T)、k(T) —— 常物性即可
- ❌ 辐射 / 对流边界 —— V1 只看脉冲内 + 短延时，可以忽略
